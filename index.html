Handout: Date/Time in SQL von Robert Schulz

Einführung

Datum und Uhrzeit spielen eine entscheidende Rolle in Datenbankanwendungen, da sie es ermöglichen, Zeitpunkte präzise festzulegen und damit beispielsweise Buchungen, Transaktionen oder Ereignisse genau zu datieren. Die korrekte Handhabung von Datum und Zeit gewährleistet nicht nur die Integrität der Daten, sondern ermöglicht auch zeitbasierte Abfragen, die für Planung, Analyse und Reporting unerlässlich sind.

1. Grundlagen

Datentypen verstehen:

  • DATE: Speichert nur das Datum (YYYY-MM-DD).
  • TIME: Speichert nur die Uhrzeit (HH:MM:SS).
  • DATETIME: Kombiniert Datum und Uhrzeit (YYYY-MM-DD HH:MM:SS).
  • TIMESTAMP: Speichert Datum und Uhrzeit, berücksichtigt Zeitzonen.
  • YEAR: Speichert ein Jahr, entweder als zwei- oder vierstellige Zahl.

Beispiel:


  
  Copy to clipboard

SELECT YEAR (CURDATE()); -- Gibt das aktuelle Jahr zurück

2. Funktionen

Standardfunktionen zur Datum- und Zeitmanipulation:

  • NOW(): Gibt das aktuelle Datum und die aktuelle Uhrzeit zurück.
  • CURDATE(): Gibt das aktuelle Datum zurück.
  • CURTIME(): Gibt die aktuelle Uhrzeit zurück.
  • DATEDIFF(date1, date2): Berechnet die Differenz in Tagen zwischen zwei Daten.
  • TIMEDIFF(time1, time2): Berechnet die Differenz zwischen zwei Uhrzeiten.
  • DATE_ADD(date, INTERVAL expr type): Fügt eine spezifische Zeitspanne zu einem Datum hinzu.

Beispiele:


  
  Copy to clipboard

SELECT NOW(); -- '2024-04-21 12:45:00'
SELECT CURDATE(); -- '2024-04-21'
SELECT CURTIME(); -- '12:45:00'
SELECT DATEDIFF('2024-12-31', '2024-01-01'); -- 365
SELECT TIMEDIFF('18:00:00', '16:00:00'); -- '02:00:00'
SELECT DATE_ADD('2024-04-21', INTERVAL 1 YEAR); -- '2025-04-21'

3. Manipulation und Anwendung

Daten- und Zeitmanipulationstechniken:

  • ADDDATE(date, INTERVAL value unit): Fügt einem Datum eine spezifische Zeiteinheit hinzu.
  • SUBDATE(date, INTERVAL value unit): Subtrahiert eine Zeiteinheit von einem Datum.
  • Verwendung von BETWEEN für Zeitintervalle.

Beispiel:


  
  Copy to clipboard

SELECT ADDDATE('2024-04-21', INTERVAL 10 DAY); -- '2024-05-01'
SELECT SUBDATE('2024-04-21', INTERVAL 3 MONTH); -- '2024-01-21'
SELECT *
FROM events
WHERE start_time BETWEEN '15:00:00' AND '17:00:00';

4. Lokalisierung

  • Speichere alle Zeitstempel in UTC, um Konsistenz zu gewährleisten.
  • Verwende CONVERT_TZ() für die Umwandlung zwischen Zeitzonen.

Beispiel:


  
  Copy to clipboard

SELECT CONVERT_TZ('2024-04-21 12:00:00', 'UTC', 'Europe/Berlin'); -- Umrechnung von UTC nach Berliner Zeit

AT TIME ZONE

AT TIME ZONE ist eine SQL-Funktion, die für die Umwandlung von Zeitstempeln zwischen verschiedenen Zeitzonen verwendet wird. Sie ist besonders nützlich in Anwendungen, die Daten von Benutzern aus verschiedenen geographischen Standorten verarbeiten, da sie sicherstellt, dass die Zeitangaben für alle Benutzer korrekt interpretiert und angezeigt werden.

Funktionsweise

AT TIME ZONE kann auf zwei Weisen verwendet werden:

  1. Konvertierung von TIMESTAMP zu TIMESTAMP WITH TIME ZONE: Dabei wird ein Zeitstempel, der ursprünglich ohne Zeitzoneninformation gespeichert wurde, mit einer spezifischen Zeitzone versehen.
  2. Konvertierung zwischen Zeitzonen: Dabei wird ein TIMESTAMP WITH TIME ZONE von einer Zeitzone in eine andere umgerechnet.

Beispiele

  • Zeitzone einem TIMESTAMP hinzufügen: 
    
  
  Copy to clipboard

    SELECT TIMESTAMP '2024-04-21 12:00:00' AT TIME ZONE 'Europe/Berlin';
  • Konvertierung zwischen Zeitzonen: 
    
  
  Copy to clipboard

    SELECT TIMESTAMP '2024-04-21 12:00:00' AT TIME ZONE 'Europe/Berlin' AT TIME ZONE 'America/New_York';

Anwendungstipps

  • Datenintegrität: Verwende AT TIME ZONE, um die Konsistenz der Zeitdaten über verschiedene Systeme und Standorte hinweg zu gewährleisten.
  • Benutzerfreundlichkeit: Passe die Anzeige von Zeitdaten an die lokale Zeitzone des Benutzers an, um Verwirrung zu vermeiden und die Benutzererfahrung zu verbessern.
  • Reporting und Analyse: Stelle sicher, dass die Berichte und Analysen die Zeitdaten korrekt nach der lokalen Zeit der Ereignisse anzeigen, insbesondere wenn Ihre Datenbank Ereignisse aus verschiedenen Zeitzonen enthält.
